# 仓库高级配置

每个仓库除了基本的地址和标签外，还支持分支选择、版本锁定和路径过滤等高级配置。

## 在 UI 中配置

在设置页面的「仓库」选项卡中，每个仓库行右侧有一个展开按钮。点击后可以看到高级配置选项。

## 配置项

### 分支（Branch）

指定要同步的 Git 分支。

- **留空** — 使用仓库的默认分支（通常是 `main` 或 `master`）
- **指定分支名** — 如 `develop`、`docs` 等

适用场景：文档在非默认分支上维护。

### 版本锁定（Pin Commit）

将同步内容固定到特定的 Git commit，不再跟随分支更新。

#### 自动锁定

1. 在 UI 中打开「锁定版本」开关
2. 此时显示「下次同步时自动锁定」
3. 执行同步后，Gleaner 会自动记录当前 commit SHA

#### 手动指定

如果已经知道 commit SHA，可以直接填入。

#### 更新到最新

点击「更新到最新」按钮，会清除锁定的 SHA，在下次同步时重新获取并锁定最新版本。

#### 解除锁定

关闭「锁定版本」开关即可恢复跟随分支最新版本。

### 路径过滤

控制只同步仓库中的特定目录。

#### 包含路径（Include Paths）

只有匹配这些前缀的 `.md` 文件才会被同步。例如：

- `docs/` — 只同步 `docs` 目录下的文件
- `wiki/zh/` — 只同步中文 wiki 目录

可以添加多条规则，满足任一即包含。

#### 排除路径（Exclude Paths）

匹配这些前缀的文件不会被同步。例如：

- `docs/internal/` — 排除内部文档
- `archive/` — 排除归档目录

排除规则在包含规则之后应用。

## YAML 格式

所有高级配置都可以在 YAML 中表达：

```yaml
repos:
  - url: my-org/knowledge-base
    label: 知识库
    branch: main
    commit: pin                    # 下次同步时自动锁定
    includePaths:
      - docs/
      - guides/
    excludePaths:
      - docs/drafts/

  - url: my-org/api-docs
    label: API 文档
    branch: develop
    commit: a1b2c3d4e5f6           # 锁定到具体 commit

  - url: my-org/notes
    label: 笔记
    # 不指定 branch/commit = 跟随默认分支最新版本
```

### commit 字段取值

| 值 | 含义 |
|----|------|
| 不填或 `latest` | 跟随分支最新版本（默认） |
| `pin` | 下次同步时自动获取并锁定 |
| `<SHA>` | 锁定到指定的 commit |

## 使用场景

| 场景 | 推荐配置 |
|------|----------|
| 只读稳定版文档 | `commit: pin` 锁定版本 |
| 跟踪开发中的文档 | `branch: develop` |
| 大仓库只看部分内容 | `includePaths` 过滤 |
| 排除草稿目录 | `excludePaths: [drafts/]` |
