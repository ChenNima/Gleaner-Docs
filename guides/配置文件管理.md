# 配置文件管理

Profile（配置文件）是 Gleaner 管理仓库列表的方式。你可以创建多个 Profile 来组织不同场景下的仓库组合。

## Profile 类型

### 本地 Profile

仓库列表以 YAML 格式存储在浏览器本地。适合个人使用。

在设置页面的「仓库」选项卡中，你可以通过表单界面添加/删除仓库，也可以切换到 YAML 编辑器直接编辑配置。

### GitHub Profile

仓库列表从一个 GitHub 仓库中的 `gleaner.yaml` 文件读取。适合团队共享同一份配置。

只需指定包含 `gleaner.yaml` 的仓库地址（如 `team-org/knowledge-base-config`），Gleaner 会自动拉取并解析配置。

## 创建 Profile

在设置页面的「配置文件」选项卡中：

1. 点击「+ 本地」或「+ GitHub」
2. 输入 Profile 名称
3. 点击保存

## 切换 Profile

有两种方式切换：

- **导航栏** — 点击品牌名旁边的 Profile 切换器
- **设置页面** — 在「配置文件」选项卡中点击目标 Profile

切换 Profile 时，Gleaner 会：

1. 中止当前正在进行的同步
2. 加载新 Profile 的仓库配置
3. 保留已缓存的共享仓库数据（如果两个 Profile 包含相同仓库）
4. 自动开始同步

## YAML 配置格式

无论是本地 Profile 的 YAML 编辑，还是 GitHub Profile 的 `gleaner.yaml`，格式都相同：

```yaml
repos:
  - url: ChenNima/Gleaner-Docs
    label: Gleaner 文档
  - url: my-org/wiki
    label: 团队 Wiki
    branch: develop
    includePaths:
      - docs/
```

完整的 YAML 字段说明请参见 [[仓库高级配置]]。

## 导入导出

你可以在「导入/导出」选项卡中：

- **导出** — 将当前 Profile 的配置下载为 `gleaner.yaml` 文件
- **从文件导入** — 上传一个 `gleaner.yaml` 文件
- **从 GitHub 导入** — 输入仓库地址直接拉取配置

这对于在不同设备之间迁移配置很有用。

## 删除 Profile

在 Profile 列表中点击删除图标。注意：

- 不能删除最后一个 Profile
- 删除活跃 Profile 会自动切换到第一个剩余的 Profile
- 删除 Profile 不会删除已缓存的文件数据
