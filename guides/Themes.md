# Themes

Gleaner offers multiple Markdown rendering themes that change how your documents look. Each theme provides a different reading experience tailored to specific preferences.

## Available Themes

| Theme | Description |
|-------|-------------|
| **GitHub** | Clean, familiar styling based on GitHub's Markdown rendering. Good default choice. |
| **Obsidian** | Styling inspired by the Obsidian note-taking app. Compact with distinctive heading styles. |
| **Academic** | Serif fonts with generous spacing, optimized for long-form reading of research-oriented content. |
| **Notion** | Minimal, modern styling inspired by Notion's clean aesthetic. |
| **Newsprint** | Classic print-style typography with a warm, paper-like feel. |

## Switching Themes

1. Look for the theme picker in the breadcrumb area above the document content
2. Select a theme from the dropdown
3. The theme is applied immediately to the current document

Your theme selection is automatically saved and persists across sessions.

## Dark / Light Mode

Gleaner supports both dark and light color modes, independent of the Markdown theme:

- Toggle between dark and light mode using the button in the top navigation bar
- The mode affects the entire UI (sidebars, navigation, backgrounds), not just the content area
- Your preference is saved automatically

## Code Highlighting

Syntax highlighting in fenced code blocks automatically adapts to both the selected theme and the dark/light mode. You don't need to configure this separately — code blocks always use colors that are readable in the current visual context.
