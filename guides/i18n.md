# Internationalization (i18n)

Gleaner's user interface supports multiple languages. You can choose between following your system language or explicitly setting a preferred language.

## Language Options

| Option | Behavior |
|--------|----------|
| **Follow system** | Gleaner detects your browser/OS language setting and uses it automatically |
| **English** | Forces the UI to English regardless of system setting |
| **Chinese** | Forces the UI to Chinese regardless of system setting |

## System Detection Logic

When set to "Follow system", Gleaner reads the browser's `navigator.language` property:

- If the value starts with `zh` (e.g. `zh-CN`, `zh-TW`, `zh`), Chinese is used
- For all other values, English is used as the default

## What It Affects

The language setting applies to **all UI elements**, including:

- Navigation bar labels
- Sidebar headers and controls
- Settings page labels and descriptions
- Search dialog placeholder text
- Status messages and sync progress
- Button labels and tooltips

**Note:** The language setting does **not** affect document content. Your Markdown files are rendered as-is in whatever language they are written in.

## Persistence

Your language preference is stored in `localStorage` and persists across browser sessions. Clearing browser data or localStorage will reset it to the default "Follow system" behavior.
