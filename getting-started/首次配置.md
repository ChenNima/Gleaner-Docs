# 首次配置

首次打开 Gleaner 时，你需要创建一个配置文件（Profile）并添加至少一个仓库。

## 创建 Profile

Gleaner 使用 **Profile（配置文件）** 来管理仓库列表。你可以创建多个 Profile 在不同的知识库之间切换。详见 [[配置文件管理]]。

首次使用时，系统会自动创建一个默认的本地 Profile。

## 添加仓库

1. 在「仓库」选项卡中，输入 `owner/repo` 格式的仓库名
2. 给它一个标签（可选），方便在 [[文件浏览|文件树]] 中辨认
3. 点击「添加」将仓库加入列表
4. 点击「保存并同步」开始下载文件

### 示例仓库

想快速体验？试试添加这个文档仓库：

```
ChenNima/Gleaner-Docs
```

## 同步过程

点击保存后，Gleaner 会：

1. 通过 GitHub API 获取仓库的文件树
2. 筛选出所有 `.md` 文件
3. 在后台逐个下载文件内容
4. 解析 [[双向链接]] 并建立索引

顶部状态栏会显示同步进度（如「同步中 5/20」）。同步完成后，你可以在 [[文件浏览|左侧边栏]] 中看到文件树。

> 没有配置 [[GitHub Token 配置|Token]] 时，API 限制为每小时 60 次请求。如果仓库文件较多，建议先配置 Token。

## 可选配置

- [[GitHub Token 配置]] — 提升 API 限额，访问私有仓库
- [[代理配置]] — 在中国大陆等地区配置 GitHub 代理
- [[仓库高级配置]] — 选择分支、锁定版本、过滤路径

## 下一步

配置完成后，回到首页开始 [[文件浏览|浏览文件]]。试试用 [[全局搜索|Cmd+K]] 搜索内容。
