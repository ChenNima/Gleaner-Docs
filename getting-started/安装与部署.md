# 安装与部署

Gleaner 是一个纯前端应用，没有后端依赖。你可以自行部署到任何静态托管服务。

## 前置要求

- [Node.js](https://nodejs.org/) 18+
- [pnpm](https://pnpm.io/) 包管理器

## 本地开发

```bash
# 克隆仓库
git clone https://github.com/ChenNima/Gleaner.git
cd Gleaner

# 安装依赖
pnpm install

# 启动开发服务器
pnpm run dev
```

开发服务器默认运行在 `http://localhost:5173`。

## 构建生产版本

```bash
pnpm run build
```

构建产物在 `dist/` 目录中，可以部署到任何静态文件服务器。

## 部署选择

| 平台 | 说明 |
|------|------|
| GitHub Pages | 免费，适合公开文档 |
| Vercel | 自动部署，自带 CDN |
| Netlify | 类似 Vercel，拖拽部署 |
| 自建 Nginx | 完全控制，适合内网使用 |

> 由于 Gleaner 使用客户端路由（React Router），部署时需要配置所有路径回退到 `index.html`。
