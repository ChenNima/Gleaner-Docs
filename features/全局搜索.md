# 全局搜索

Gleaner 提供类似 VS Code / Linear 的 Cmd+K 命令面板，在所有已缓存的文件中进行即时搜索。

## 打开搜索

有三种方式打开搜索框：

| 方式 | 说明 |
|------|------|
| `Cmd+K` / `Ctrl+K` | 键盘快捷键（推荐） |
| 点击导航栏搜索图标 | 鼠标操作 |
| 移动端：菜单中的搜索图标 | 触屏操作 |

## 搜索范围

搜索会在当前 [[配置文件管理|Profile]] 中所有已同步仓库的文件中查找匹配项：

- **文件标题** — 标题匹配的结果排在前面
- **文件内容** — 全文搜索，不区分大小写

> 搜索是在本地缓存中进行的，无需网络请求，速度极快。

## 搜索结果

每条搜索结果显示：

- **文件标题** — 加粗显示
- **所属仓库** — 以标签形式显示
- **内容片段** — 约 120 字符的上下文，匹配文字高亮
- **文件路径** — 完整路径方便定位

## 键盘导航

在搜索框打开时，你可以完全使用键盘操作：

| 按键 | 功能 |
|------|------|
| `↑` `↓` | 在结果中上下移动 |
| `Enter` | 打开选中的文件 |
| `Esc` | 关闭搜索框 |

选中的结果会自动滚动到可见区域。

## 搜索技巧

- 输入文件名的一部分即可快速定位
- 搜索特定术语来找到包含该内容的所有文件
- 搜索是增量的 — 每次输入都会实时更新结果
- 清空搜索框可以重新开始

## 与其他功能的关系

- 搜索到的文件可以直接跳转到 [[文件浏览]] 页面
- 在 [[文件浏览]] 中看到的 [[双向链接]] 引用了未知文件？用搜索确认它是否存在
- 搜索范围受 [[多仓库管理]] 中当前 Profile 的仓库列表影响
- 更多键盘操作见 [[快捷键]]
