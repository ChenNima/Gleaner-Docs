# 文件浏览

Gleaner 采用类似 Obsidian 的三栏布局来浏览和阅读 Markdown 文件。

## 布局概览

| 栏位 | 位置 | 内容 |
|------|------|------|
| 文件树 | 左侧边栏 | 可展开/折叠的仓库文件树 |
| 内容区 | 中间 | Markdown 渲染与面包屑导航 |
| 右侧面板 | 右侧边栏 | 链接标签页（反向链接 + 外链）和图谱标签页（局部图谱） |

## 文件树（左侧边栏）

左侧边栏显示所有已同步仓库的文件树。

- **展开/折叠** — 点击文件夹图标来展开或折叠目录
- **排序规则** — 目录排在前面，文件排在后面，均按字母顺序排列
- **多仓库分区** — 当配置了多个仓库时，每个仓库作为顶层分区显示，各自拥有独立的可折叠文件树
- **全部折叠** — 使用侧边栏顶部的全部折叠按钮可一键收起整个文件树

点击 `.md` 文件即可在内容区中打开。

## 内容区

中间栏渲染选中的 Markdown 文件，完整支持：

- **标准 Markdown** — 标题、段落、列表、引用块、图片、表格
- **GitHub 风格 Markdown (GFM)** — 任务列表、删除线、自动链接、表格
- **语法高亮代码块** — 通过 highlight.js 实现语言特定的代码高亮
- **内联 HTML** — 原始 HTML 元素按原样渲染
- **Wikilinks** — `[[target]]` 和 `[[target|alias]]` 链接渲染为可点击的链接，在 Gleaner 内部导航

### 面包屑导航

内容上方显示面包屑导航栏，展示从仓库根目录到当前文件的完整路径。每个路径段都可以点击，方便快速向上导航。

## 右侧面板

右侧边栏提供两个标签页：

- **链接** — 显示[反向链接](./Backlinks.zh.md)（链接到当前文件的其他文件）、外发链接（当前文件链接到的其他文件）以及外部链接
- **图谱** — 以当前文件为中心显示局部[知识图谱](./Knowledge%20Graph.zh.md)，展示其直接关联

## 侧边栏控制

- **切换按钮** — 顶部导航栏中有按钮可以切换两个侧边栏的显示/隐藏
- **拖动调整宽度** — 在桌面端，拖动侧边栏边缘可以调整其宽度
- **移动端覆盖层** — 在小屏幕上，侧边栏以覆盖层形式打开，点击外部区域可关闭
