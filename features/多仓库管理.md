# 多仓库管理

Gleaner 支持同时接入多个 GitHub 仓库，让你可以将来自不同项目的文档统一在一个阅读界面中。

## 添加多个仓库

在设置页面的「仓库」选项卡中，你可以添加任意数量的仓库。每个仓库需要：

- **仓库地址** — `owner/repo` 格式
- **标签** — 用于在 [[文件浏览|文件树]] 中区分不同仓库

## 文件树分组

多个仓库的文件会在左侧边栏中分组显示：

```
▾ Gleaner-Docs          ✓ 15
  ▾ getting-started
    快速开始.md
    安装与部署.md
  ▾ features
    ...
▾ My-Notes              ✓ 42
  ▾ daily
    ...
```

每个仓库是一个独立的可折叠区域，旁边显示同步状态和文件数。

## 跨仓库链接

[[双向链接|Wikilinks]] 支持跨仓库解析。如果一个仓库中的文件引用了 `[[某个主题]]`，Gleaner 会：

1. 先在**当前仓库**中查找匹配的文件
2. 如果没找到，再搜索**其他已同步的仓库**

这意味着你可以在不同仓库的文件之间建立引用关系。

## 图谱中的多仓库

在 [[知识图谱|全局图谱]] 中，不同仓库的文件使用不同颜色标注。你还可以：

- 在图谱页面按仓库过滤显示
- 通过颜色快速识别文件来源
- 看到跨仓库的链接关系

## 同步管理

每个仓库独立同步，互不影响：

- 一个仓库同步失败不会阻止其他仓库
- 每个仓库显示独立的同步状态和进度
- 点击「刷新」会重新同步所有仓库

## 使用 Profile 组织

如果你有不同的工作场景（比如「工作」和「个人」），可以创建多个 [[配置文件管理|Profile]]，每个 Profile 包含不同的仓库组合。通过导航栏的 Profile 切换器快速切换。

## 高级配置

每个仓库可以独立配置：

- **分支** — 指定要同步的分支
- **版本锁定** — 固定到某个 commit
- **路径过滤** — 只同步特定目录

详见 [[仓库高级配置]]。
