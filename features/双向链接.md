# 双向链接

双向链接（Backlinks）是 Gleaner 的核心功能之一，借鉴自 [[快速开始|Obsidian]] 的设计理念。它让你的笔记之间形成一张知识网络，而不仅仅是单向的层级结构。

## Wikilink 语法

在 Markdown 中使用双方括号创建链接：

```markdown
这篇笔记引用了 [[文件浏览]]。
```

渲染后，`[[文件浏览]]` 会变成一个可点击的链接，点击后跳转到对应的文件。

### 别名

如果你想显示不同的文字，可以使用管道符：

```markdown
查看 [[文件浏览|侧边栏功能]] 了解更多。
```

渲染为：查看 [[文件浏览|侧边栏功能]] 了解更多。

### 链接解析规则

Gleaner 按以下顺序解析 wikilink 目标：

1. **文件标题匹配** — 先在当前仓库中查找标题匹配的文件
2. **文件名匹配** — 再按文件名（不含 `.md` 后缀）匹配
3. **跨仓库匹配** — 如果当前仓库没有匹配，搜索其他已同步仓库

> 同名文件的情况下，**当前仓库优先**。这在 [[多仓库管理]] 场景下很重要。

### 链接状态

- **已解析链接** — 目标文件存在，显示为正常链接样式
- **未解析链接** — 目标文件不存在，显示为灰色虚线样式

## 标准 Markdown 链接

除了 wikilinks，Gleaner 也会解析标准的 Markdown 链接：

```markdown
[查看配置指南](../guides/配置文件管理.md)
```

相对路径会根据当前文件位置自动解析。

## 反向链接面板

在右侧面板的「链接」选项卡中，你可以看到：

### 反向链接（Backlinks）

所有引用了当前文件的其他文件列表。例如，当你阅读这篇「双向链接」文档时，右侧会显示所有通过 `[[双向链接]]` 引用了这篇文档的页面。

### 出链（Outgoing Links）

当前文件中所有指向其他文件的链接。

### 外部链接

当前文件中指向外部网站的链接，如 `[GitHub](https://github.com)`。

## 在知识图谱中查看

双向链接关系也会在 [[知识图谱]] 中可视化呈现。每条 wikilink 或 Markdown 链接都会在图谱中表现为一条连线。

## 小贴士

- 建议多使用 wikilinks 来建立笔记间的联系
- 给每篇笔记起一个清晰的标题，这样 wikilink 解析更准确
- 查看 [[知识图谱]] 可以发现你可能忽略的关联
- 本文档就大量使用了 wikilinks，打开右侧面板看看效果
