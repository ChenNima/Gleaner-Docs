# 多仓库管理

Gleaner 支持同时连接多个 GitHub 仓库，让你可以从不同来源构建统一的知识库。

## 添加多个仓库

在设置页面中，你可以添加任意数量的仓库。每个仓库以 `owner/repo` 格式指定，可以分别设置标签、分支、版本锁定和路径过滤规则。

## 文件树组织

当多个仓库同步后，左侧边栏按仓库分组显示文件：

- 每个仓库作为顶层分区显示，带有名称和文件计数标签
- 状态指示器显示每个仓库的同步状态（已同步、同步中、错误）
- 每个分区可以独立展开或折叠

## 跨仓库 Wikilink 解析

Wikilinks 在所有已同步仓库中全局解析。当你写 `[[某个笔记]]` 时：

1. Gleaner 首先搜索**当前仓库**（包含链接文件的仓库）
2. 如果未找到匹配项，则搜索**所有其他已同步仓库**

这意味着你可以使用相同的 `[[wikilink]]` 语法在不同仓库的文件之间无缝链接。当前仓库优先机制确保当存在重复标题时，优先匹配本地文件。

## 图谱可视化

在[知识图谱](./Knowledge%20Graph.zh.md)中，每个仓库被分配不同的颜色。这使得你可以方便地：

- 查看哪些仓库贡献了最多内容
- 识别跨仓库连接
- 逐个切换仓库的显示/隐藏以聚焦视图

## 独立同步

每个仓库独立同步：

- 你可以一次触发所有仓库的同步，也可以单独重新同步某个仓库
- 文件树 SHA 比较是按仓库进行的，未变化的仓库会被完全跳过
- 如果某个仓库同步失败（如触及速率限制），其他仓库不受影响

## 使用 Profile 组织

使用 [配置文件管理](../guides/Profiles.zh.md) 为不同场景组织不同的仓库集合。例如，你可以为工作相关的仓库创建一个 Profile，为个人知识库创建另一个。切换 Profile 可以即时加载不同的仓库配置。
