# 双向链接

双向链接是 Gleaner 的核心功能之一，实现了 Markdown 文件之间的双向导航。Gleaner 同时追踪 wikilinks 和标准 Markdown 链接，构建完整的链接索引。

## Wikilink 语法

Gleaner 支持与 Obsidian 相同的 wikilink 语法：

- **基本链接** — `[[target]]` 创建指向名为 "target" 的文件的链接
- **别名链接** — `[[target|显示文本]]` 创建显示为「显示文本」但导航到 "target" 的链接

Wikilinks 在内容区渲染为可点击的链接，样式与标准 Markdown 链接有所区分。

## 链接解析规则

当你写下 `[[target]]` 时，Gleaner 按以下优先级进行解析：

1. **标题匹配** — 查找标题（第一个 `# 标题`）与 "target" 匹配的文件
2. **文件名匹配** — 查找文件名（去掉 `.md` 扩展名后）与 "target" 匹配的文件
3. **跨仓库解析** — 如果当前仓库中没有找到匹配项，Gleaner 会搜索所有其他已同步的仓库

当多个仓库中存在多个匹配项时，**当前仓库**（包含链接文件的仓库）中的文件优先。

## 链接状态

链接通过视觉样式指示其解析状态：

- **已解析链接** — 显示为标准可点击链接，点击可导航到目标文件
- **未解析链接** — 以特殊样式显示（通常是变暗或带虚线下划线），表示未找到匹配文件

## 标准 Markdown 链接

除了 wikilinks，Gleaner 也会追踪标准 Markdown 链接：

```markdown
[显示文本](./path/to/file.md)
```

这些链接会被解析并纳入链接索引，因此它们会像 wikilinks 一样出现在反向链接面板和知识图谱中。

## 右侧面板：链接标签页

查看文件时，右侧面板的**链接**标签页显示三个区域：

- **反向链接** — 通过 wikilinks 或标准链接*指向*当前文件的文件
- **外发链接** — 当前文件*指向*的其他文件
- **外部链接** — 指向知识库外部的 URL（如 `https://...`）

每个条目显示文件标题和周围上下文的摘要。

## 可视化

所有追踪的链接都会在[知识图谱](./Knowledge%20Graph.zh.md)中可视化呈现，文件显示为节点，链接显示为边。这提供了笔记之间互联关系的全局视角。

## 使用建议

- **大量使用 wikilinks** — 笔记之间的链接越多，反向链接和图谱就越丰富
- **使用清晰的标题** — 由于解析优先检查标题，清晰且唯一的一级标题（`# 标题`）使链接更可靠
- **检查未解析链接** — 定期检查未解析的链接，修正错别字或创建缺失的文件
