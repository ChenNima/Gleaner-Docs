# 全局搜索

Gleaner 提供快速的 Cmd+K 风格命令面板，用于在所有已同步文件中搜索。搜索完全基于本地缓存运行，因此结果即时呈现。

## 打开搜索

有三种方式打开搜索对话框：

| 方式 | 说明 |
|------|------|
| **Cmd+K** (Mac) / **Ctrl+K** (Windows/Linux) | 在应用任何位置使用的键盘快捷键 |
| **导航栏按钮** | 点击顶部导航栏中的搜索图标 |
| **移动端菜单** | 在移动端汉堡菜单中点击搜索选项 |

## 工作原理

搜索查询同时匹配每个文件的**标题**（第一个 `# 标题`）和**完整内容**。匹配不区分大小写，完全基于本地 IndexedDB 缓存运行，无需网络请求。

结果随输入实时出现，没有延迟。

## 搜索结果

每条结果显示：

- **标题** — 文件的一级标题或文件名
- **仓库标签** — 彩色标签，指示文件所属的仓库
- **摘要** — 匹配位置前后约 120 个字符的上下文，匹配文本高亮显示
- **文件路径** — 在仓库内的相对路径

结果按相关性排序，标题匹配的权重高于内容匹配。

## 键盘导航

搜索对话框打开后，可以完全使用键盘操作：

| 按键 | 操作 |
|------|------|
| **上/下方向键** | 在结果列表中移动选择 |
| **Enter** | 打开选中的文件 |
| **Esc** | 关闭搜索对话框 |

## 使用技巧

- **增量搜索** — 开始输入后结果即时更新，无需按 Enter 触发搜索
- **支持部分匹配** — 不需要输入完整的标题或单词，部分字符串也能匹配
- **跨仓库搜索** — 结果包含所有已同步仓库的文件，不仅限于当前仓库
